# ⚛️ What is React?

**React** is a JavaScript library for building user interfaces (UIs).

It was created by **Meta** (formerly Facebook).

You use React to build:

* Web apps
* Dashboards
* Single Page Applications (SPAs)
* Interactive interfaces

If HTML is the *structure* and CSS is the *style*, React controls the **behavior and dynamic updates**.

---

# 🧠 Why React Exists

Before React:

* Updating the DOM manually was messy
* Code became hard to manage
* Big apps were painful to maintain

React solves this by:

* Breaking UI into **components**
* Automatically updating only what changes
* Making UI predictable

---

# 🧩 Core Concept 1: Components

Everything in React is a **component**.

Think of components like LEGO blocks 🧱

Example:

* Navbar → component
* Sidebar → component
* Button → component
* Entire page → component

```jsx
function Welcome() {
  return <h1>Hello, world!</h1>;
}
```

That’s literally a React component.

---

# 🔁 Core Concept 2: State

**State = data that can change.**

Example:

* Counter number
* User input
* Logged-in status
* Theme (dark/light)

```jsx
const [count, setCount] = useState(0);
```

When state changes → React re-renders the component automatically.

This is the magic part ✨

---

# 📨 Core Concept 3: Props

**Props = data passed from one component to another.**

Parent → Child communication.

```jsx
function Greeting(props) {
  return <h1>Hello {props.name}</h1>;
}
```

Props are:

* Read-only
* Used for customization

---

# 🌳 Virtual DOM (Important Concept)

React uses something called the **Virtual DOM**.

Instead of updating the real browser DOM directly:

1. React creates a virtual copy
2. Compares changes
3. Updates only what changed

This makes React **fast and efficient**.

---

# 🎣 Hooks (Modern React)

Hooks let you use features like state in functional components.

Most important ones:

* `useState()` → manage state
* `useEffect()` → run side effects (API calls, timers)
* `useContext()` → global state
* `useRef()` → access DOM elements

Hooks made React much cleaner than older class components.

---

# 🛠 React Ecosystem

React itself only handles UI.

For full apps you usually add:

| Purpose          | Tool                               |
| ---------------- | ---------------------------------- |
| Routing          | React Router                       |
| State Management | Redux / Context                    |
| API Calls        | fetch / axios                      |
| Styling          | CSS / Tailwind / Styled Components |
